# 画像管理方針

ASAKA Photography LP では、画像を「世界観を見せる素材」だけでなく、「予約前の不安を減らす情報」として扱います。

## 基本方針

- スマホで見たときに主役が伝わる画像を優先する
- 表情、関係性、空気感、距離感が伝わる写真を優先する
- 画像は必要なサイズに圧縮・最適化してから使う
- LP本文の流れが固まるまでは、大量の画像を入れない
- 元データや既存資料は参照のみとし、無断で移動・編集しない

## `public/` に置くもの

`public/` は、ファイル名とURLを固定してそのまま公開したい静的ファイルに使います。

例:

- OGP画像
- favicon
- robots.txt
- 外部サービスから直接参照する必要がある画像
- 差し替えURLを固定したい画像

推奨配置:

```text
public/
  images/
    ogp/
    icons/
```

## `src/assets/` に置くもの

`src/assets/` は、Astroの画像最適化やコンポーネント内で扱う画像に使います。

例:

- LP本文内で使う写真
- セクションごとに読み込むビジュアル
- コンポーネントに紐づく画像

推奨配置:

```text
src/assets/
  photos/
    hero/
    gallery/
    proof/
  ui/
```

## 命名ルール

画像ファイル名は、役割と内容がわかる英小文字のケバブケースにします。

例:

- `hero-family-morning.jpg`
- `gallery-maternity-window-01.jpg`
- `proof-natural-smile-01.jpg`

## 選定観点

- 最初の1枚で、ASAKA Photography らしい空気が伝わるか
- 予約前の不安が減るか
- 撮影体験のやさしさが伝わるか
- スマホで見たときに人物や感情が埋もれないか
- CTAへ進む流れを邪魔しないか

## 避ける画像

- 暗すぎて内容が伝わりにくい画像
- 情報量が多く、スマホで主役がわからない画像
- 世界観は良いが、予約検討者の安心につながりにくい画像
- CTAやテキストの視認性を大きく下げる画像

## 運用メモ

LP本文の構成が固まるまでは、画像は仮置きしすぎないこと。

写真は美しさだけでなく、検討者が「この人に任せても大丈夫そう」と感じられるかを基準に選びます。
